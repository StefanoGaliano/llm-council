# LLM Council — Blueprint

> Generated by The Architect on 2026-06-25
> Archetype: API / Backend Service (adapted for CLI orchestration)

---

## 1. Project Overview

### Vision

The **LLM Council** is a command-line tool that runs a structured, multi-agent *adversarial* evaluation of early-stage B2B / enterprise software concepts. A single human types in a concept; the tool spins up **1 Researcher agent + 7 specialized Council personas** governed by a visible **Master Orchestrator**, and drives them through a strict **5-phase protocol** designed to simulate the friction of reality *before* capital is committed.

Unlike a "brainstorm with an LLM" tool, the Council is built to *kill* bad ideas: enforced role separation eliminates confirmation bias, every claim must trace back to the Researcher's timestamped State Document, and the final verdict (`GO` / `NO-GO` / `CONDITIONAL GO`) is delivered by a silent Bayesian synthesis agent that scores five weighted dimensions. The orchestrator is always visible — announcing phases, enforcing turn order, and interrupting unsupported claims and persona breaches in real time.

It is a CLI for a single power user (you), packaged so it can later be shared with others. It uses **real** data grounding via free-tier integrations (Tavily/Brave web search, GitHub API) and **gracefully degrades** where no free API exists (Crunchbase/PitchBook/LinkedIn → web-search-derived, low-confidence-flagged).

### Goals

- Faithfully implement the v2.0 system prompt: 1 Researcher + 7 personas + Master Orchestrator, 5-phase protocol, all role constitutions and flags.
- Make the protocol **enforced by code**, not by hoping the LLM follows instructions — deterministic phase/turn sequencing with LLM judgment only where genuine judgment is required (flagging).
- Deliver a "watch the council debate live" terminal experience (Ink TUI) with streaming output, phase headers, inline orchestrator flags, and a running cost meter.
- Produce durable, resumable run artifacts (State Document, transcript, verdict, metadata) on disk.
- Keep per-run cost predictable and visible via tiered models + prompt caching.

### Success Metrics

- A full evaluation of a real concept completes end-to-end and writes a coherent `verdict.md` with a defensible verdict and weighted score matrix.
- The orchestrator demonstrably catches at least the two flag classes (`UNSUPPORTED CLAIM`, `PERSONA BREACH`) on adversarial test inputs.
- A typical tiered run lands in the low single-digit USD range, shown accurately in `run.json`.
- A run can be resumed and re-grounded via the Pivot Protocol without corrupting state.

## 2. Tech Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Runtime | Node.js 20+ | Modern ESM, native fetch, fits your existing Node/ruflo ecosystem |
| Language | TypeScript (strict) | Type safety across the agent/state contracts; Zod-inferred types |
| CLI parsing | Commander | Minimal, ubiquitous; thin layer that mounts the Ink app |
| Live UI | Ink (React for CLI) + ink-spinner | Per-agent live panes, streaming tokens, inline flag banners, cost footer |
| LLM SDK | @anthropic-ai/sdk | First-class streaming, tool use, **prompt caching** (`cache_control`) |
| Models | Opus 4.8 (`claude-opus-4-8`) + Sonnet 4.6 (`claude-sonnet-4-6`) | Tiered: Opus for Orchestrator checks + Decider; Sonnet for personas + Researcher |
| Web search | Tavily (primary) + Brave Search (fallback) | Both free-tier; LLM-optimized results; auto-failover |
| Code signals | Octokit (GitHub REST) | Stars, last commit, open issues, contributor count — free with a PAT |
| Validation | Zod | Env config + the Researcher's output JSON schema (single source of truth for types) |
| Persistence | Filesystem (per-run folder) | No DB needed for a single-user CLI; artifacts are portable markdown/JSON |
| Testing | Vitest | Fast, ESM-native, great mocking for the LLM/tool boundaries |
| Lint/format | ESLint + Prettier | Standard hygiene |
| Package manager | pnpm | Per project convention |
| Distribution | npm bin (`council`) | `pnpm link` / `npm i -g` for sharing later |

**Models policy:** Always default to the latest Claude models. IDs are pinned in `src/config/models.ts` so a future bump is a one-line change. Do **not** invent model IDs.

## 4. Data Model

This is a CLI with no database. The "data model" is the in-memory + on-disk run state. All shapes are defined in `src/types/index.ts` and the Researcher output is additionally a Zod schema in `src/researcher/schema.ts`.

### Core types

**Run** (in-memory state, mirrored to `run.json`)
| Field | Type | Notes |
|-------|------|-------|
| id | string | `<concept-slug>-<ISO-timestamp>` |
| concept | string | Raw concept text the user submitted |
| createdAt | string | ISO timestamp |
| phase | `0\|1\|2\|3\|4` | Current phase pointer (for resume) |
| stateDocument | StateDocument | The Researcher's grounding (see below) |
| transcript | Turn[] | Ordered record of every agent turn |
| flags | Flag[] | Every orchestrator flag raised |
| objectionLedger | string[] | The Client's running unresolved concerns |
| conflictMap | Conflict[] | Output of Phase 1 |
| verdict | Verdict \| null | Populated in Phase 4 |
| cost | CostLedger | Token + USD totals |

**StateDocument** (mirrors System Prompt Section 2 — both `state.md` and `payload.json`)
| Field | Type | Notes |
|-------|------|-------|
| conceptSummary | string | |
| timestamp | string | ISO; appears in `state.md` header |
| competitorMatrix | Competitor[] | name, stage, totalFundingUsd, coreDifferentiator, githubStars, lastCommitDaysAgo, recentSignal, confidence |
| openSourceAlternatives | OssAlt[] | project, stars, lastCommitDaysAgo, maturityLevel, displacementRiskScore (1–5) |
| marketSizing | { tam, sam, somYear1 } | each: figure + source + year |
| technicalInfraBenchmarks | Benchmark[] | latency, compute cost/unit, uptime SLA |
| regulatoryLandscape | RegItem[] | frameworks + enforcement precedents |
| talentSignal | TalentItem[] | roles, supply/demand, notable movement |
| overwriteLog | OverwriteEntry[] | `[MEMORY_OVERWRITE]` events: field, reason, oldValue, newValue, timestamp |

**Turn** (one agent speaking)
| Field | Type | Notes |
|-------|------|-------|
| phase | number | |
| round | number \| null | Phase 2 only |
| personaId | PersonaId | `businessMan`, `informatic`, … |
| content | string | The agent's full output |
| flags | Flag[] | Flags raised against *this* turn |
| resubmission | boolean | True if this is a post-flag re-submission |
| usage | { inputTokens, outputTokens, cachedTokens, usd } | |

**Flag**
| Field | Type | Notes |
|-------|------|-------|
| type | `UNSUPPORTED_CLAIM \| PERSONA_BREACH \| CIRCULAR_REASONING` | |
| personaId | PersonaId | |
| detail | string | What triggered it; for unsupported claims, the offending claim text |
| resolved | boolean | True if agent sourced/retracted on re-submission |

**Verdict**
| Field | Type | Notes |
|-------|------|-------|
| evidenceSynthesis | { claim, tag }[] | tag ∈ `SUPPORTED \| UNSUPPORTED \| CONTESTED` |
| conflictResolutions | { conflict, favoredPersona, rationale }[] | |
| scoreMatrix | DimensionScore[] | 5 dimensions, weight 20% each, score 0–100, ≥2 cited quotes |
| decision | `GO \| NO_GO \| CONDITIONAL_GO` | |
| conditions | string[] | If CONDITIONAL_GO: measurable milestones |
| killCondition | string \| null | If NO_GO: primary kill + which agent established it |
| unresolvedObjections | string[] | Leftover ledger items = automatic risk flags |
| nextAction | string | One sentence, specific, actionable |

### On-disk artifacts (per run folder)

| File | Purpose |
|------|---------|
| `state.md` | Human-readable State Document (System Prompt §2 layout) |
| `payload.json` | Zod-validated Researcher JSON (System Prompt §2 mandatory output) |
| `transcript.md` | Full debate, phase by phase, with flags inline |
| `verdict.md` | Decider's final judgment + score matrix + verdict |
| `run.json` | Full Run object (resume source of truth) |

## 5. API Design

There is no HTTP API. The "API" is (a) the **CLI surface** and (b) the **internal agent/tool contracts**.

### CLI surface

| Command | Description |
|---------|-------------|
| `council run "<concept>"` | Start a new evaluation. `--fast` skips interactive confirmation; `--model-tier all-opus\|all-sonnet\|tiered`. |
| `council run` | No arg → interactive concept prompt (ink-text-input). |
| `council resume <run-id>` | Resume an interrupted run from its saved phase. |
| `council resume <run-id> --pivot "<change>"` | Pivot Protocol: re-run Researcher, then resume. |
| `council list` | List past runs with verdict + cost. |
| `council config` | Set/verify `ANTHROPIC_API_KEY`, `TAVILY_API_KEY`, `BRAVE_API_KEY`, `GITHUB_TOKEN`. |

### Internal contracts (the critical interfaces)

**`runAgent(persona, context) → AsyncIterable<string> + Turn`**
- Input: persona constitution, current phase/round, the State Document (injected, cached), the transcript slice the persona is allowed to see.
- Output: streamed tokens for the TUI + a finalized `Turn`.

**Orchestrator Check (`checks.ts`) — the heart of the system**
- Runs **on every turn** (per your decision). After an agent finalizes a Turn, an **Opus** call receives: the agent's output, the agent's role constitution, and the full `payload.json`.
- It returns a **structured tool-call** (not prose) of shape:
  ```json
  { "flags": [ { "type": "UNSUPPORTED_CLAIM", "detail": "claims competitor X raised $40M — not in State Document", "quote": "..." } ] }
  ```
- If `flags` is non-empty: the orchestrator emits the visible banner (`[ORCHESTRATOR FLAG: ...]`), and the agent gets **one** re-submission with the flags appended to its prompt ("source these or retract"). After one retry, unresolved flags are recorded and the turn stands.
- Detection rules the check enforces:
  - `UNSUPPORTED_CLAIM`: any named company/statistic/product/funding figure not present in `payload.json`.
  - `PERSONA_BREACH`: role-constitution violation (e.g., Informatic proposes a solution, Decider speaks before Phase 4 or uses a quality adjective, Business Man proposes a fix).
  - `CIRCULAR_REASONING`: argument that restates its conclusion as its premise.

**Researcher tool loop (`researcher.ts`)**
- Agentic loop with two tools exposed to Claude: `web_search(query)` and `github_lookup(owner/repo | search)`.
- Loops until the model emits a final structured payload validated by `schema.ts`. On validation failure → one corrective re-prompt with the Zod error.
- `state.md` is rendered from the validated payload.

## 6. Frontend Architecture

The "frontend" is the **Ink terminal UI**.

### Screens / phases (driven by `Run.phase`)

| Phase | UI |
|-------|------|
| Concept entry | `ConceptPrompt` (if no concept arg) |
| Phase 0 | Researcher progress: live tool calls (search/github), then State Document summary + integrity check result |
| Phase 1 | Four sequential `AgentPane`s (Business Man → Informatic → Financial Man → Ethicist), then the Conflict Map |
| Phase 2 | Three rounds; `AgentPane` per turn; `FlagBanner` inline; `ObjectionLedger` panel updates live; Claim Scorecard after each round |
| Phase 3 | Feynman audit: single `AgentPane` re-explanation + Client's insight/gap determination |
| Phase 4 | `ScoreMatrix` + verdict block |
| Always | `CostFooter` pinned at bottom |

### Component hierarchy

```
<App>                          # subscribes to orchestrator events
  <PhaseHeader phase round />
  <AgentPane persona streaming />     # one per active turn
    <FlagBanner flags />              # rendered when orchestrator flags the turn
  <ObjectionLedger items />          # Phase 2+
  <ScoreMatrix verdict />           # Phase 4
  <CostFooter ledger />             # always
```

### State management

- The **orchestrator is the source of truth** and runs the protocol as an async generator/event emitter. The Ink `<App>` subscribes to orchestrator events (`phase:start`, `turn:token`, `turn:end`, `flag`, `ledger:update`, `cost:update`, `verdict`) and re-renders.
- The TUI is a *pure view* — it never drives protocol logic. This keeps the protocol testable headless (the orchestrator runs identically with or without Ink).
- Persistence is written by the orchestrator on every `turn:end` so the run is always resumable.

## 7. Design System

Terminal aesthetic — defined as a small theme module (`src/ui/theme.ts`) using Ink/chalk color names + hex where the terminal supports truecolor.

### Colors (semantic, terminal palette)

| Role | Color | Usage |
|------|-------|-------|
| Orchestrator | `#A78BFA` (violet) | Phase headers, orchestrator voice |
| Flag / breach | `#F87171` (red) | `[ORCHESTRATOR FLAG]` banners |
| Supported evidence | `#34D399` (green) | `[SUPPORTED]` tags, GO verdict |
| Contested | `#FBBF24` (amber) | `[CONTESTED]`, CONDITIONAL GO, low-confidence data |
| Unsupported | `#F87171` (red) | `[UNSUPPORTED]`, NO-GO verdict |
| Persona accent | per-persona (see below) | Agent name + pane border |
| Muted | `#6B7280` (gray) | Timestamps, token counts, secondary text |
| Background | terminal default | Do not force a background |

**Per-persona accents:** Decider violet, Business Man red, Marketing Man cyan, Financial Man green, Informatic blue, Client gold, Ethicist magenta, Researcher gray.

### Typography

| Role | Treatment |
|------|-----------|
| Phase headers | Bold + underline, uppercase, persona/orchestrator color |
| Agent name | Bold, persona accent, with emoji glyph (⚖️ 💼 📣 📊 💻 👑 🔬) |
| Body | Terminal default weight; wrap at terminal width |
| Flags | Bold red, boxed (ink box border) |
| Tables (matrix/scorecard) | Monospace alignment via `ink-table` |

### Spacing & layout

- One blank line between turns; boxed panes for active agent output.
- Cost footer pinned bottom, full-width rule above it.
- Max render width = terminal columns; tables degrade gracefully on narrow terminals.

### Component style

Clean, information-dense, no gratuitous animation. The only motion is the spinner during tool calls / model streaming. Flags must be impossible to miss (red box). The vibe: a serious war-room transcript, not a chatbot.

## 8. Authentication & Authorization

No user accounts. "Auth" = third-party API credentials, loaded and validated at startup.

### Credentials

| Variable | Required | Source |
|----------|----------|--------|
| `ANTHROPIC_API_KEY` | Yes | console.anthropic.com |
| `TAVILY_API_KEY` | Yes (search primary) | app.tavily.com |
| `BRAVE_API_KEY` | Optional (search fallback) | brave.com/search/api |
| `GITHUB_TOKEN` | Recommended | github.com → Settings → Developer settings → PAT (read-only, public repos) |

### Loading & validation

- `src/config/env.ts` loads from process env + `.env` + optional `~/.councilrc`, validates with Zod, and **fails fast** with a friendly message + the `council config` hint if a required key is missing.
- Search degrades: if Tavily fails/quota-exhausted → Brave; if both absent → hard error (the Researcher cannot ground without search).
- GitHub degrades: if no token → unauthenticated API (60 req/hr) with a warning; GitHub fields tagged lower confidence if rate-limited.
- Paid sources (Crunchbase/PitchBook/LinkedIn): **never** required. The Researcher derives those signals from web search and tags them `confidence: "low"`.

## 9. Build Order

> This is the critical section. Build in this order; each step is independently runnable/testable.

**Step 1 — Scaffolding.**
`pnpm init`; install TypeScript, tsx, Vitest, ESLint, Prettier. Configure `tsconfig.json` (strict, ESM, `@/` → `src/`), `vitest.config.ts`, `eslint.config.js`. Add `package.json` `bin` field (`council` → `dist/index.js`) and scripts (`dev`, `build`, `test`, `lint`). Create the full empty directory tree.

**Step 2 — Config & types.**
Build `src/config/models.ts` (pinned model IDs `claude-opus-4-8` / `claude-sonnet-4-6` + pricing table), `src/config/env.ts` (Zod env validation, fail-fast), and `src/types/index.ts` (all shared types from Section 4). Write `.env.example`. Test: env validation rejects missing keys.

**Step 3 — LLM client + cost.**
`src/llm/client.ts`: Anthropic wrapper with streaming and `cache_control` support; a single `complete({ model, system, messages, tools?, stream })` surface. `src/llm/cost.ts`: token accounting → USD using the pricing table, including cached-token discount. Test cost math against known token counts with `tests/helpers/mockAnthropic.ts`.

**Step 4 — Researcher schema + State Document.**
`src/researcher/schema.ts` (Zod for the mandatory output JSON, incl. `confidence` on degraded fields and `overwriteLog`). `src/researcher/stateDocument.ts`: render validated payload → `state.md`; implement `[MEMORY_OVERWRITE]` diff logic. Test: schema accepts the canonical example payload; overwrite logic logs field/reason.

**Step 5 — Researcher tools.**
`src/researcher/tools/webSearch.ts` (Tavily → Brave failover) and `tools/github.ts` (Octokit: stars, last commit days ago, open issues, contributors). Test with mocked HTTP. Each tool returns normalized, source-tagged results.

**Step 6 — Researcher agent loop.**
`src/researcher/researcher.ts`: agentic loop exposing the two tools to Sonnet; loops to a final payload; validates with Zod; one corrective retry on validation failure. Renders `state.md` + writes `payload.json`. Emits progress events. Test the loop with a mocked tool-using model.

**Step 7 — Persona constitutions + generic runner.**
`src/agents/constitutions/*` (verbatim role mandates, forbidden behaviors, output requirements from System Prompt §3). `src/agents/registry.ts` (persona → constitution + model + phase rules). `src/agents/runAgent.ts`: assembles system prompt (constitution, cached) + user prompt (State Document cached + allowed transcript slice), streams output, returns a `Turn`.

**Step 8 — Orchestrator Check (flags).**
`src/orchestrator/checks.ts`: Opus structured tool-call returning `{ flags: [...] }`; implements the three detection rules (unsupported claim vs `payload.json`, persona breach vs constitution, circular reasoning). Wire the one-retry re-submission contract. Test: feed a turn that cites a non-existent competitor → expect `UNSUPPORTED_CLAIM`; feed the Decider speaking in Phase 2 → expect `PERSONA_BREACH`.

**Step 9 — Orchestrator state machine.**
`src/orchestrator/phases.ts` (phase + turn-order tables, incl. Phase 2's 3 rounds and which personas speak when; Decider silent until Phase 4; Ethicist speaks Phase 2 R2 + Phase 3). `src/orchestrator/orchestrator.ts`: drives Phase 0→4, invokes Researcher, runs each turn through the Check, builds `conflictMap.ts` (Phase 1) and `scorecard.ts` (per round), maintains the Objection Ledger, emits events, persists after each turn. Implement Phase 0 data-integrity gate (repeat Phase 0 if incomplete). Test the full pipeline headless with a mocked model.

**Step 10 — The Decider (Phase 4).**
`src/agents/decider.ts`: consumes full transcript + ledger + conflict map → evidence synthesis with `[SUPPORTED]/[UNSUPPORTED]/[CONTESTED]` tags, conflict resolutions, weighted 5×20% score matrix (each score cites ≥2 proceeding quotes), verdict + conditions/kill-condition, unresolved objections, one-sentence next action. Opus. Validate output against the `Verdict` type.

**Step 11 — Persistence + resume + pivot.**
`src/persistence/*`: run-folder lifecycle, writers for `transcript.md`/`verdict.md`/`run.json`, loaders for resume. Implement `council resume` (restart from saved phase) and `--pivot` (halt → re-run Researcher with overwrite logging → resume). Test: save mid-Phase-2, reload, continue.

**Step 12 — Ink TUI.**
Build `src/ui/App.tsx` + components (Section 6) subscribing to orchestrator events. Then `src/index.ts` + `src/commands/*` (Commander) to mount it. `ConceptPrompt` for no-arg interactive entry; `council config`/`list`. Verify the live debate renders, flags banner in red, cost footer updates.

**Step 13 — Hardening + docs.**
Graceful degradation paths (search failover, GitHub rate-limit warnings, low-confidence tagging), the Token Context Rule (compressed State Document when context is large), `--model-tier` flag, README with setup + sample run, and the target-project `CLAUDE.md`. Final pass: `pnpm build && pnpm test`.

## 12. Deployment Strategy

### Distribution
Not a server — distributed as an npm CLI. Local dev via `pnpm dev`; build to `dist/` and `npm link` (or `npm i -g .`) to expose the `council` binary. To share later: publish to npm (`council` or scoped `@you/council`) or send a tarball (`npm pack`).

### Build
`tsc` (or `tsup` if you want a single bundled binary) → `dist/`. `package.json` `bin: { "council": "dist/index.js" }` with a `#!/usr/bin/env node` shebang.

### CI/CD
Optional GitHub Actions: on push run `pnpm lint && pnpm test && pnpm build`. No deploy step until/unless you publish.

### Environments
Single environment (the user's machine). Keys live in `.env` or `~/.councilrc`. `runs/` is gitignored. No staging/prod split needed.

## 13. Testing Strategy

### Unit Tests (Vitest)
- `cost.ts`: USD math incl. cached-token discount.
- `schema.ts`: accepts canonical payload, rejects malformed, applies `confidence` defaults.
- `stateDocument.ts`: `[MEMORY_OVERWRITE]` diff/log correctness.
- `phases.ts`: turn order correctness (Decider silent 1–3, Ethicist only R2 + Phase 3).

### Integration Tests
- `checks.test.ts`: flag detection for `UNSUPPORTED_CLAIM`, `PERSONA_BREACH`, `CIRCULAR_REASONING` using a deterministic mocked Opus response.
- Full orchestrator pipeline headless with `tests/helpers/mockAnthropic.ts` (no network): Phase 0→4 produces a valid `Verdict` and writes all artifacts.
- Resume/pivot: save mid-run, reload, continue without state corruption.

### Manual / E2E
- One real end-to-end run against live APIs on a known concept; eyeball `verdict.md` and the cost in `run.json`. Use `/verify` or `/run` skill to drive the live run.

# LLM Council

A CLI that runs a multi-agent adversarial evaluation (1 Researcher + 7 personas + a visible Orchestrator) of B2B software concepts through a strict 5-phase protocol, then issues a GO / NO-GO / CONDITIONAL GO verdict.

## Commands

Node 20 + TypeScript (strict) + Commander + Ink (React-for-CLI) + @anthropic-ai/sdk (Opus 4.8 + Sonnet 4.6) + Tavily/Brave search + Octokit + Zod. Filesystem persistence. Vitest. pnpm.

## Architecture

### Directory Structure
- `src/orchestrator/` — The state machine. Deterministic phase/turn sequencing + per-turn Orchestrator Check (LLM flagging). The source of truth for a run.
- `src/agents/` — Persona constitutions + generic runner; `decider.ts` is the Phase 4 synthesis.
- `src/researcher/` — Tool-using grounding agent → `state.md` + `payload.json` (Zod-validated).
- `src/llm/` — Anthropic wrapper (streaming, cache_control), model routing, cost accounting.
- `src/ui/` — Ink components; a PURE VIEW that subscribes to orchestrator events.
- `src/persistence/` — Run-folder lifecycle, resume/pivot.
- `src/config/` — Zod env validation + pinned model IDs/pricing.
- `src/types/` — Shared contracts.

### Data Flow
CLI command → orchestrator runs Phase 0 (Researcher: search/github tools → State Document) → Phases 1–4 (each persona turn → Orchestrator Check → flags/re-submit → persist) → Decider verdict. The Ink UI only renders orchestrator events; it never drives logic. Every turn is persisted to `runs/<id>/run.json` for resumability.

### Key Patterns
- **Protocol is code, judgment is LLM.** Phase/turn order is deterministic; only flagging and content generation call the model.
- **State Document is the single source of truth.** Every persona prompt injects `payload.json` (with prompt caching). Unsupported-claim detection compares claims against it.
- **One retry on flags.** Flagged turns get exactly one re-submission, then the flag stands and is logged.
- **Graceful degradation.** Tavily→Brave failover; GitHub unauth fallback; paid data sources (Crunchbase/PitchBook/LinkedIn) derived from web search and tagged `confidence: "low"`. Never require a paid API.
- **Cost is tracked everywhere.** Token + USD per call, running total in the TUI footer and `run.json`.

## 16. Reglas No Negociables

1. **The protocol is enforced by code.** Phase sequencing, turn order, and which persona may speak when are deterministic — never delegated to an LLM's good intentions. The model only generates content and emits flags.
2. **No claim without grounding.** Every named company/statistic/funding figure an agent uses must exist in the Researcher's `payload.json`, or the Orchestrator Check flags `[UNSUPPORTED CLAIM]` and forces source-or-retract (one retry).
3. **Persona constitutions are inviolable.** Role breaches (Decider speaking early or judging, Informatic/Business Man proposing solutions) must be flagged `[PERSONA BREACH]`.
4. **The orchestrator is always visible.** Phase headers, flags, Conflict Map, Claim Scorecards, and the Objection Ledger are surfaced in the UI and written to the transcript.
5. **Free-tier only; degrade, never fake.** Paid sources (Crunchbase/PitchBook/LinkedIn) are derived from web search and tagged `confidence: "low"`. No fabricated precision.
6. **Cost is visible and bounded.** Tiered models + prompt caching by default; per-call and total token/USD tracked and shown.
7. **Runs are durable and resumable.** Persist after every turn; support `resume` and the `--pivot` Pivot Protocol (re-ground via Researcher with `[MEMORY_OVERWRITE]` logging before resuming).
8. **TypeScript strict, no `any`.** Validate all boundaries with Zod. Files under 500 lines. All external calls injectable + mocked in tests.
